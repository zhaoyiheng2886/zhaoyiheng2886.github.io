本文简单探究一下语音生成模型的发展，主要看了三篇文章WaveNet(2016), VALL-E(2023), VALL-E 2(2024).
## 从 WaveNet 到 VALL-E 2：语音生成模型在生成什么

语音生成这件事，表面上看是在“从文本生成语音”，但如果追问一步：语音究竟是什么东西，事情就没那么简单了。

从物理上看，语音就是空气振动形成的连续波；从数字系统看，语音就是一串离散采样点；从感知上看，语音又不只是“把字念出来”，还包括音色、韵律、停顿、情绪、环境混响这些说不清但又非常重要的东西。WaveNet、VALL-E、VALL-E 2 这三篇文章，本质上就在回答同一个问题：我们到底应该在哪个层面上生成语音，才能既自然、又高效、还稳。 WaveNet 选择直接生成原始波形；VALL-E 选择先把语音压成离散 codec token 再生成；VALL-E 2 则是在后者路线上继续解决重复和长序列建模问题。
### 一、WaveNet：大道至简

如果我们完全不耍花样，那么语音生成最直接的办法是什么？答案其实很朴素：既然语音在计算机里就是一串采样值，那我就一个采样点一个采样点地往后生成。 这就是 WaveNet 的基本想法。论文把它定义成一个对 raw audio waveform 做概率建模的自回归模型：当前采样点的分布，条件于前面所有采样点。

这个想法有一种“所见即所得”的美感。你不需要先设计一堆中间表示，也不需要假装自己知道哪些声学特征最重要，而是直接面对最终信号本身。这样的好处也很明显：它离真实波形最近，因此有机会把最细微的语音纹理也学进去。 论文报告说，WaveNet 用在 TTS 时，自然度显著超过当时最好的 parametric 和 concatenative 系统，英语和中文都如此。

但问题也恰恰出在这里。因为“离最终信号最近”，也意味着它在做最细粒度的建模。16kHz 音频一秒就有 16000 个采样点，24kHz 就更多。你让模型逐点自回归，相当于让它在一个极长、极细、局部变化极其频繁的序列上做 next-step prediction。训练还能并行一些，推理时却天然要一步一步来，于是质量虽然高，速度却很难受。换句话说，WaveNet 的困难不是它不会生成，而是它太认真了，认真到每一粒沙子都要自己摆。

所以，WaveNet 留给后人的启发并不只是“raw waveform 可以生成得很好”，更重要的是另一半：如果你坚持在波形层面生成，那么生成问题会被展开成一个极长、极底层的序列建模问题。 这条路是通的，但代价很大。
### 二、VALL-E：让预测目标从连续到离散

VALL-E 最本质的改动，不是网络换成了更潮的 Transformer，而是它不再直接生成波形，而是先把语音变成 neural codec 的离散 token，再把 TTS 改写成条件语言建模问题。论文对此说得很明确：**它使用现成 neural audio codec 导出的 discrete codes，并将 TTS 视为 conditional language modeling，而不是连续信号回归**。

这一步为什么关键？因为它相当于承认了一件事：波形虽然是真实语音的最终载体，但未必是最适合生成模型直接操作的表示。 就像图像生成后来普遍转向 latent space 一样，语音生成也可以先做一层“压缩和重参数化”，把原本密密麻麻、局部抖动剧烈的连续信号，整理成一串更短、更结构化的**离散**符号。于是，模型不再面对“下一个采样值是多少”这种近乎像素级的问题，而是面对“下一块语音积木是什么”这样更抽象的问题。

如果说 WaveNet 的思想是“忠实地雕刻每一个波形点”，那么 VALL-E 的思想更像是“先把语音切成可以复用的声学词元，再按语境去续写”。这一改写带来两个后果。第一，问题形式立刻更像语言模型了，因此 Transformer 这套东西可以更自然地拿来用；第二，模型开始有机会学到一种更高层的“说话方式延续能力”，而不仅是局部波形拟合能力。论文中的结果就是：只给 3 秒未见说话人的录音作为 acoustic prompt，VALL-E 就能直接合成该说话人的高质量语音，而且保留其情绪和声学环境。

### 三、好的离散化为什么会让问题变简单

很多人第一次看到 VALL-E，都会有一个直觉：预测离散 token，当然比直接预测连续波形简单。这种直觉基本没错，但如果只停在这里，就还是浅了一层。

更准确地说，离散本身没有魔法，真正起作用的是“合适的离散化”。 科学空间里有一类文章经常强调这一点：模型的关键往往不在于你喊出了什么新名词，而在于你有没有把问题变换到一个更适合求解的空间里。VALL-E 背后的 neural codec，就扮演了这个“换空间”的角色。它不是随便量化一下，而是先用一个学出来的编解码器，把语音压到一个既尽量保真、又比原波形紧凑得多的离散表示上。于是后面的 LM 不是在做粗暴分类，而是在一个经过表征学习整理过的空间里建模。

为什么这种改写通常会更好学？因为连续波形的“正确答案”过于细碎，而且常常不是唯一的。哪怕是同一句话，稍微不同的相位、停顿、细节震动，都可能对应非常不一样的波形，但听感上却差不多。直接回归这种目标，模型往往容易被大量低层细节绑架。相反，离散 codec token 把这些细节折叠到一个更高层的词表里之后，模型只需要先把大方向走对，再由 codec decoder 负责还原细节。这种分工更像“规划”和“渲染”的分离。

所以，VALL-E 的成功并不只是“连续变离散”这么简单，而是：先用 codec 把连续语音整理成更有结构的离散语言，再用语言模型去生成这门语言。 从这个意义上说，VALL-E 更像是在发明一种“语音的书写系统”。

### 四、VALL-E 2：方向已经对了，接下来是把坑填平

通常来说，一条路线最难的不是提出，而是做成熟。VALL-E 证明了 neural codec LM 这条路能走通，但真正落到生成过程里，问题还会冒出来。最典型的就是重复、卡顿、长句不稳。VALL-E 2 的有趣之处在于，它没有推翻前作，而是非常务实地做了两件事：一个针对解码时的重复问题，一个针对序列太长的问题。论文把它们分别叫做 Repetition Aware Sampling 和 Grouped Code Modeling。

先说 Repetition Aware Sampling。名字看着挺花，其实直觉很简单：模型在采样时，不能只看“当前哪个 token 概率最大”，还得看“我是不是已经在重复自己了”。 这有点像语言模型生成文本时，有时会掉进局部高概率循环，一直说同一套话。语音里这个问题更明显，因为重复几个 token，听感上立刻就像卡带一样。VALL-E 2 通过在 nucleus sampling 里把“解码历史中的重复情况”也纳入考虑，来稳定解码并避免 infinite loop。

再说 Grouped Code Modeling。这个点更体现出工程上的成熟：即便用了 codec token，语音序列依然很长。 长序列一长，训练难、推理慢、局部错误积累都跟着来。VALL-E 2 的做法是把 codec codes 组织成 group，以更高效的方式建模，等于在不明显损失表达力的前提下，把序列长度压短了。这不仅提升速度，也缓解了长序列建模本身的困难。

你会发现，到了这一代，问题已经不再是“要不要在 token 空间生成”，而是“既然已经在 token 空间生成了，怎么让它真正像一个成熟的生成系统一样稳定工作。” 这其实很像很多模型路线的发展规律：第一篇解决“能不能做”，第二篇解决“做得好不好”，第三篇开始解决“什么时候会翻车，以及怎么不翻车”。VALL-E 2 在 LibriSpeech 和 VCTK 上报告了在人类自然度、鲁棒性、说话人相似度上的进一步提升，并把自己定位到 human parity。

### 五、再多一点点

如果沿着这条脉络继续往后想，会自然冒出两个问题。

第一个问题是：codec 本身是不是最优表示？ 如果 codec 仍然损失了一些微妙的韵律或情感结构，那么 VALL-E 路线的上限就会被 codec 限制。第二个问题是：离散 token 真的是唯一出路吗？近年的一些文章其实也在讨论连续输入、连续建模以及“世界模型”式理解，强调离散化虽然方便自回归，但也可能引入信息损失。类似的张力，在语音里同样存在。

截至目前，语音模型已经进入百花齐放的时代了，codec token LM已经不是唯一答案，Flow matching/diffusion TTS表现也很亮眼，2026的工作前沿问题已经变成，模型可不可以听，想，说。这和文本 LLM 的发展很像：先把生成做好，再把理解、推理、交互并进去。
